# Rust reference equivalence report (Mission 5, H36)

Status: **research report**. Not a certified revision; does not modify `ITD V29.18`.
Evidence class: **software-equivalence**. Reproduce with `cargo test --workspace` in
`itd-rs/` and `pytest tests/test_rust_equivalence.py`.

## Question (H36)

Does the initial Rust implementation reproduce the authoritative Python reference within
locked tolerances?

## What was implemented

A pure-std Rust workspace (`itd-rs/`, no external crates → builds/tests fully offline):

* **itd-field** — typed row-major `Field2D`, periodic central-difference vorticity,
  deterministic (fixed-order, single-accumulator) reductions.
* **itd-diagnostics** — enstrophy, vorticity RMS, and localization on that subset.
* **itd-result** — a product result contract with an `is_bare_alarm()` guard mirroring
  the Python `ITDAnalysisResult`, plus deterministic hand-rolled JSON.

`#![forbid(unsafe_code)]` throughout; `cargo fmt`/`clippy -D warnings` clean.

## Equivalence method

The committed fixture `itd-rs/fixtures/diagnostics.txt`
(generated by `tools/rust/generate_diagnostics_fixture.py`) is the **shared oracle**:

* `cargo test` (`rust_matches_python_reference`) checks the Rust computation reproduces
  the fixture's expected values within the **preregistered relative tolerance 1e-9**;
* `tests/test_rust_equivalence.py` checks the Python reference recomputes the same
  expected values from the fixture input.

Both sides matching the fixture establishes Python↔Rust equivalence on the subset.

## H36 classification: **supported within tested scope**

The Rust implementation reproduces the Python reference for the diagnostics subset
within 1e-9 (relative), deterministically, offline. The result contract's bare-alarm
guard is unit-tested on both sides.

## Explicitly unsupported (stated, not hidden)

* The Rust crate reproduces a **periodic-central-difference subset**, **not** the
  certified `ITD V29.18` signature (finite-boundary operator + `structural_metrics`),
  nor the 3D channels. Those remain **Python-only**.
* Rust is a **reference**, never the scientific oracle; the Python V29.18 core remains
  authoritative.
* Streaming, OOD scoring, and the full pipeline are not ported.

## Limitations

One diagnostic subset, 2D, one fixture; tolerance (not bitwise) equivalence, because
NumPy's pairwise summation and Rust's sequential sum differ at ~1e-15. The bounded
scope is deliberate: external validation is not complete, so the full solver/signature
port is intentionally deferred.
